## Задание: Создание задачи

### TC001: Успешное создание задачи

**Серьезность:** Высокий  
**Приоритет:** Высокий  
**Тип тестирования:** Позитивный  
**Предусловия:** Открыт сайт таск-трекера

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Нажать кнопку "Создать задачу" | Открывается форма создания задачи с полями: Название, Описание, Проект, Приоритет, Статус, Исполнитель |
| 2 | Заполнить поле "Название" текстом "Тестовая задача" | Поле заполняется введенным текстом |
| 3 | Заполнить поле "Описание" текстом "Описание тестовой задачи" | Поле заполняется введенным текстом |
| 4 | Нажать на поле "Проект" и выбрать первый проект из выпадающего списка | Поле заполняется выбранным проектом |
| 5 | Нажать на поле "Приоритет" и выбрать приоритет "Low" из выпадающего списка | Поле заполняется значением "Low" |
| 6 | Нажать на поле "Исполнитель" и выбрать первого исполнителя из выпадающего списка | Поле заполняется выбранным исполнителем |
| 7 | Нажать кнопку "Создать" | - Форма закрывается<br>- Задача создается и появляется в списке задач<br>- При поиске "Тестовая задача" задача отображается в результатах |

---

### TC002: Валидация обязательных полей при создании задачи

**Серьезность:** Средний  
**Приоритет:** Высокий  
**Тип тестирования:** Негативный  
**Предусловия:** Открыт сайт таск-трекера

| Шаг | Сценарий | Действие | Ожидаемый результат |
|-----|----------|----------|---------------------|
| 1 | Без названия | - Оставить поле "Название" пустым<br>- Заполнить поля: Проект (первый), Приоритет (Low), Исполнитель (первый)<br>- Нажать "Создать" | Кнопка "Создать" неактивна (имеет класс Mui-disabled) |
| 2 | Без проекта | - Заполнить поле "Название": "Задача без проекта"<br>- Оставить поле "Проект" пустым<br>- Заполнить Приоритет (Low), Исполнитель (первый)<br>- Нажать "Создать" | Кнопка "Создать" неактивна |
| 3 | Без приоритета | - Заполнить поле "Название": "Задача без приоритета"<br>- Заполнить Проект (первый)<br>- Оставить поле "Приоритет" пустым<br>- Заполнить Исполнитель (первый)<br>- Нажать "Создать" | Кнопка "Создать" неактивна |
| 4 | Без статуса | - Заполнить все обязательные поля: Название "Задача без статуса", Проект (первый), Приоритет (Low), Исполнитель (первый)<br>- Нажать "Создать" | Задача успешно создается и отображается при поиске "Задача без статуса" |
| 5 | Без исполнителя | - Заполнить поля: Название "Задача без исполнителя", Проект (первый), Приоритет (Low)<br>- Оставить поле "Исполнитель" пустым<br>- Нажать "Создать" | Кнопка "Создать" неактивна |

### TC016: Поэтапное редактирование всех полей задачи через карточку

**Серьезность:** Высокий  
**Приоритет:** Высокий  
**Тип тестирования:** Позитивный  
**Предусловия:** Открыт сайт таск-трекера, в системе существует задача "Адаптация карточки для мобильных устройств"

**Шаги выполнения:**

#### Этап 1: Редактирование описания задачи
| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1.1 | В поле поиска ввести "Адаптация карточки для мобильных устройств" | Задача отображается в результатах поиска |
| 1.2 | Нажать на задачу "Адаптация карточки для мобильных устройств" | Открывается форма редактирования задачи |
| 1.3 | В поле "Описание" выделить весь текст и удалить его | Поле "Описание" становится пустым |
| 1.4 | Ввести новый текст "Обновленное описание задачи" | Поле заполняется новым текстом |
| 1.5 | Нажать кнопку "Обновить" | Форма закрывается, изменения сохраняются |
| 1.6 | Повторно найти и открыть задачу "Адаптация карточки для мобильных устройств" | В поле "Описание" отображается текст "Обновленное описание задачи" |
| 1.7 | Закрыть карточку задачи клавишей ESC | Форма редактирования закрывается |

#### Этап 2: Редактирование приоритета задачи
| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 2.1 | Найти и открыть задачу "Адаптация карточки для мобильных устройств" | Открывается форма редактирования задачи |
| 2.2 | Нажать на поле "Приоритет" | Открывается выпадающий список с вариантами приоритетов |
| 2.3 | Выбрать приоритет "Medium" | Поле "Приоритет" заполняется значением "Medium" |
| 2.4 | Нажать кнопку "Обновить" | Форма закрывается, изменения сохраняются |
| 2.5 | Повторно найти и открыть задачу "Адаптация карточки для мобильных устройств" | В поле "Приоритет" отображается значение "Medium" |
| 2.6 | Закрыть карточку задачи клавишей ESC | Форма редактирования закрывается |

#### Этап 3: Редактирование статуса задачи
| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 3.1 | Найти и открыть задачу "Адаптация карточки для мобильных устройств" | Открывается форма редактирования задачи |
| 3.2 | Нажать на поле "Статус" | Открывается выпадающий список с вариантами статусов |
| 3.3 | Выбрать статус "InProgress" | Поле "Статус" заполняется значением "InProgress" |
| 3.4 | Нажать кнопку "Обновить" | Форма закрывается, изменения сохраняются |
| 3.5 | Повторно найти и открыть задачу "Адаптация карточки для мобильных устройств" | В поле "Статус" отображается значение "InProgress" |
| 3.6 | Закрыть карточку задачи клавишей ESC | Форма редактирования закрывается |

#### Этап 4: Редактирование исполнителя задачи
| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 4.1 | Найти и открыть задачу "Адаптация карточки для мобильных устройств" | Открывается форма редактирования задачи |
| 4.2 | Нажать на поле "Исполнитель" | Открывается выпадающий список с исполнителями |
| 4.3 | Выбрать второго исполнителя из списка | Поле "Исполнитель" заполняется выбранным значением |
| 4.4 | Нажать кнопку "Обновить" | Форма закрывается, изменения сохраняются |

| 4.6 | Закрыть карточку задачи клавишей ESC | Форма редактирования закрывается |
